SECTION 5: REVIEW PROCESS
═══════════════════════════════════════════════════════════════════════════════

Follow this exact process for every wireframe review:

STEP 1: INITIAL SCAN (2 minutes)
─────────────────────────────────
Quick check for obvious violations:
- Border radius visible anywhere?
- Correct number of HUB tiles?
- 2×2 grid structure?
- Colors roughly correct?

If major violations obvious → Prepare for reject/revise

STEP 2: CONSTRAINT AUDIT (10 minutes)
─────────────────────────────────────
Systematically check each of 7 constraints:
- Document every violation found
- Note specific location (line number, selector)
- Capture current vs required values

STEP 3: DIMENSIONAL SCORING (10 minutes)
────────────────────────────────────────
Score across 5 dimensions:
- Assign 1-10 score for each
- Document rationale for each score
- Calculate weighted total

STEP 4: DECISION (3 minutes)
────────────────────────────
Based on constraint audit and scoring:

IF weighted score ≥ 9 AND zero constraint violations:
  → DECISION: PASS
  → Proceed to user presentation

IF weighted score 7-8 OR 1-2 minor violations:
  → DECISION: REVISE
  → Provide specific corrections to P1

IF weighted score < 7 OR major/fundamental violations:
  → DECISION: REJECT
  → Require complete rewrite

STEP 5: DOCUMENTATION (10 minutes)
──────────────────────────────────
Compile review report with all findings
