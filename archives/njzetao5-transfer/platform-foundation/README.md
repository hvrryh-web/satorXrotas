# PixelOffice Platform Foundation

This folder implements a concrete startup-ready blueprint for building:
- WebApp (project and planner experience)
- Website (marketing and docs platform)
- Browser extension
- Pixel sprite assistant UX
- AI-agent simulation and project management flows

Use the documents in this folder to scaffold the production repository structure, standards, and onboarding process.

## Document Index
- `PRODUCT_SCOPE.md` - Product boundaries and feature map
- `REPO_STRUCTURE.md` - Core repo domains and scaffold layout
- `ENGINEERING_STANDARDS.md` - Branching, review, CI, security, release standards
- `QUICKSTART.md` - Local setup, environment template, service contracts
- `PIXELOFFICE_AGENT.md` - Modular sprite + planner + simulation design
- `ONBOARDING.md` - Onboarding-first developer experience
- `ROADMAP.md` - Phased milestones and acceptance criteria
- `WIKI_OPERATIONS_PROTOCOL.md` - Wiki-ready operations protocol, stack gaps, and agent learning workflow
- `AI_DEVELOPER_CLOUD_SERVICES_PLAYBOOK.md` - AI developer enablement, autonomy policy, and cloud/service blueprint
- `REVIEW_SUITE_RECOMMENDATIONS.md` - OKR/CRIT/PRD/ARD and gap-based recommendations
- `CODEBASE_RESEARCH_AND_SERVICE_EXAMPLES.md` - Researched patterns and explicit service code examples for planned features
- `IMPLEMENTATION_BACKLOG.md` - Execution backlog with phased API/UI deliverables and definition-of-done gates

## Scaffold Seed
The `scaffold/` directory provides a ready-to-adopt folder skeleton for future app/service implementation.
