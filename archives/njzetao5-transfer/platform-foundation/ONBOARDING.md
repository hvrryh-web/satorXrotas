# Onboarding-First Developer Experience

## Goals
- New contributor reaches first merged PR quickly.
- Contributors learn architecture through guided tasks.
- AI-assisted workflow is understandable and safe by default.

## Onboarding Flow
1. **Day 0**: Environment setup and repo walkthrough
2. **Day 1**: First issue with starter task label
3. **Day 2+**: Guided contribution to one domain (app, service, extension)

## Required Onboarding Assets
- Architecture map for domains and data flow
- Local setup quickstart with troubleshooting
- Contributor rules (branching, testing, PR quality)
- Sample tasks by difficulty (starter/intermediate/advanced)

## Suggested Starter Task Categories
- UI sprite state tweak
- Planner validation rule update
- API endpoint contract extension
- Browser extension capture enhancement
